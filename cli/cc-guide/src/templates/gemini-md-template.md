# Gemini CLI 协作指南

> 版本: 1.0.0 | 更新日期: 2026-03-16

本文档为 Gemini CLI 提供核心概念和最佳实践的快速参考。

---

## 核心概念

| 机制 | 位置 | 用途 |
|------|------|------|
| **GEMINI.md** | 项目根目录 | 项目上下文和指令 |
| **AGENTS.md** | 项目根目录 | 代理行为配置 |
| **Skills** | `.gemini/skills/` | 可预加载的知识模块 |
| **Commands** | `.gemini/commands/` | 用户调用的提示模板 |
| **Hooks** | `.gemini/hooks/` | 事件触发的脚本 |

---

## 模型选择

| 模型 | 适用场景 | 特点 |
|------|----------|------|
| `gemini-2.0-flash` | 快速响应任务 | 高速、低成本 |
| `gemini-2.0-pro` | 复杂推理任务 | 高质量、多模态 |
| `gemini-ultra` | 最复杂任务 | 最强能力 |

---

## Prompting 技巧

### 技巧 1: 明确角色
```
作为 [角色]，请 [任务]。
约束条件：[约束]
期望输出：[格式]
```

### 技巧 2: 分步引导
```
请按以下步骤完成任务：
1. 分析...
2. 设计...
3. 实现...
4. 验证...
```

### 技巧 3: 提供示例
```
参考示例：
输入: [示例输入]
输出: [示例输出]

现在请处理：[实际输入]
```

## 上下文管理

| 场景 | 建议 |
|------|------|
| 大型项目 | 将详细内容放入 docs/ 目录 |
| Monorepo | 每个子目录一个 GEMINI.md |
| 频繁切换任务 | 使用 /clear 重置上下文 |

## 最佳实践

1. **GEMINI.md 控制在 200 行以内**
2. **使用清晰的文件结构**
3. **明确指定输出格式**
4. **提供充分的上下文**
5. **验证关键输出**
